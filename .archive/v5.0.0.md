# Danbooru Mobile Note Assist v5.0.0 — popover가 진짜 노트 에디터가 되었다

v4.x의 popover는 textarea와 ✔ / ✖ / 🗑 세 버튼이 전부였다. 굵게, 색깔, 크기, 한자에 후리가나 — 이런 건 raw markup으로 직접 타이핑해야 했고, 결과가 어떻게 보일지는 ✓ Confirm으로 서버에 올린 다음에야 확인할 수 있었다. 이번 v5.0.0은 그 popover를 본격 markup 에디터로 다시 만들었다.

major 버전을 4에서 5로 올린 건 backwards-incompatible 변경이 있어서가 아니다 — 키보드 단축키, 제스처, API 흐름은 모두 그대로다. 다만 popover 안의 사용자 노출 면이 확 늘어났고, minor라기엔 변화 폭이 너무 컸다. v3 → v4가 force-quit recovery 한 가지였던 것에 비하면 이번 변화가 훨씬 넓다.

## Markup이 popover 안으로

popover 옆에 **Aa** 사이드 버튼이 새로 생겼다. 텍스트를 선택하고 누르면 style sub-popover가 옆으로 펼쳐진다. Bold / Italic / Underline / Strike / sub / sup / tn / code / `<a>` / ruby 태그 버튼, Text / Stroke / BG 색상 picker, Size / Font dropdown이 거기에 모두 들어 있다. 각 버튼은 선택한 텍스트만 wrap하고 그 텍스트를 다시 선택 상태로 되돌려놓기 때문에, B 누르고 I 누르면 자연스럽게 안쪽으로 nesting된다. 이미 적용된 스타일은 버튼이 활성 표시되고, 그 활성 버튼을 다시 누르면 그 한 layer만 풀린다.

색상 picker는 14개 Material 톤 swatch + HEX 입력. Stroke는 거기에 두께 (1 / 2 / 3 px)와 사방 체크박스를 갖춘 Advanced 섹션이 더 있어서 외곽선을 세밀하게 잡을 수 있다. Size는 −2부터 +3까지 6 단계, Font는 Danbooru wiki에 정의된 12개 폰트가 — 각 옵션이 자기 폰트로 미리 보여서 dropdown만 봐도 어떤 느낌인지 짐작이 된다.

`<a href>`와 `<ruby><rt>`처럼 부가 입력값이 필요한 두 태그는 작은 inline modal로 처리했다. 링크는 Danbooru 도메인 prefix를 자동으로 잘라내서 internal link가 상대 경로로 들어간다. ruby는 confirm 후 선택 영역이 `{한자}<rt>{독음}</rt>` 전체로 옮겨져서, 그 위에 다른 스타일을 또 올릴 수 있다.

매번의 스타일 토글마다 textarea 상태가 snapshot되기 때문에, popover의 ↶ 버튼이 한 번에 하나의 스타일만 깔끔히 되돌린다. raw 키보드 입력은 여전히 textarea의 native undo가 처리하니까 ↶가 글자 단위로 채워지지 않는다.

## Preview / History / view help

popover 헤더 왼쪽의 **👁 Preview** 토글은 현재 본문을 Danbooru sanitizer로 round-trip해서 결과 HTML을 보여준다. commit 전에 "다른 사용자한테는 어떻게 보일까"를 그 자리에서 확인할 수 있다. 헤더 오른쪽 **view help** 링크는 Danbooru 노트 wiki를 새 탭으로 연다.

popover 하단에 **📜 History** 버튼이 4번째 액션으로 추가됐다. 서버에 이미 저장된 노트라면 그 노트의 version history 페이지가 새 탭으로 열린다. 누가 언제 어떻게 고쳤는지 빠르게 확인할 수 있고, soft-delete된 노트에서도 활성이다 — 삭제된 노트의 과거 변경도 의미 있는 기록이라.

## Danbooru native 도구와의 충돌 방지

이전엔 Danbooru 자체 Translate 모드나 노트 편집 dialog를 켠 상태에서도 우리 floating button이 그대로 떠 있어서 두 UI가 같은 화면에서 부딪쳤다. 이번부터는 native UI가 활성화되는 순간을 감지해서 우리 button이 자동으로 숨는다. 반대 방향도 — 우리가 active mode일 때 native의 N 키나 Translate 사이드 링크를 누르면 차단하고 짧은 toast로 알린다. 두 도구가 같은 페이지에서 사이좋게 공존하는 게 목적이지, 의도하지 않은 동작이 silent하게 일어나선 안 된다.

## Pre-release audit으로 잡은 것들

기능 작업이 끝난 뒤 release 직전에 보안 / runtime race / architecture 세 영역을 별도 audit으로 돌렸다. 거기서 release blocker 5건과 그 외 9건의 fix가 모두 들어갔다.

가장 위험했던 건 Confirm batch 송신 도중 error modal이 떠 있는 사이 폰을 백그라운드로 보내면 부분적으로만 적용된 상태가 draft로 저장돼서, 다음에 페이지를 다시 열었을 때 이미 서버에 들어간 변경이 또 보내지는 경우. confirm pipeline의 lock을 modal 대기 구간까지 넓혀서 그 사이의 force-quit save를 차단하도록 했다. popover를 ✖ 또는 outside-tap으로 dismiss했을 때 ↶ 한번 누르면 사용자가 cancel한 markup이 부활하던 결함도 잡았다. cancel 시점에 그 노트의 'text' snapshot을 정리하지 않고 있어서 그랬다.

한국어 / 일본어 / 중국어 IME로 입력하다가 Ctrl+Enter나 Esc를 눌러 IME에 commit / cancel 신호를 보내려는 순간, 그 키가 그대로 popover의 ✔ / dismiss로 라우팅돼서 미완성 조합이 commit되거나 입력 박스가 닫히던 결함도 있었다. composition 진행 중 키 이벤트는 IME에 양보하도록 가드를 넣었다.

이외에 링크 URL과 ruby reading의 client-side defense-in-depth escape, draft 스키마의 per-entry validation, 4개 picker outside-tap suppress flag의 TTL safety net, ruby unwrap의 multi-rt 처리, race condition 정리, type cast 안전성 강화 등이 release 전에 모두 처리됐다. 자세한 건 CHANGELOG의 Phase 5-h 섹션 참고.

## 작은 폴리시들

stroke picker 안에 transparent swatch와 별도로 "Remove stroke" 텍스트 버튼이 같이 있어서 같은 일을 두 가지 방식으로 할 수 있던 어색함은 transparent swatch 한 가지로 정리했다. `<a>` 버튼은 ruby 버튼처럼 같은 버튼 다시 누르면 modal이 닫히는 toggle 동작을 새로 갖췄다.

v4.1 사이클의 manual check 중 발견된 force-quit recovery 관련 세 가지 regression — `hasContentToSave`의 over-eager save, `applyDraftSnapshot`의 render race, `beforeunload` prompt와 draft 충돌 — 도 이번 사이클에서 모두 hotfix로 들어갔다.

## 다음 cycle

v5.x backlog에는 block-level layout 일괄 검토 (Align / center / blockquote / ul-li / h1 등), color picker의 HEX alpha channel, stroke advanced 상태 restore, 4-picker cycle refactor 같은 안정화 항목들이 줄지어 있다. 이번 사이클에서 굵직한 markup 도구 골격이 들어갔으니, 다음은 그 위의 폴리시 + 안정화 쪽으로 갈 예정이다.

## 인스톨 / 업데이트

기존 v4.x 사용자는 별도 작업이 필요 없다. Tampermonkey가 다음 자동 업데이트 체크 때 v5.0.0을 가져온다. 새로 설치하려면 README의 인스톨 링크를 따라가면 된다. 모든 키보드 단축키와 제스처, popover 흐름은 v4.x와 동일하니 새 markup 도구를 익히는 것 외에 적응할 게 없다.

popover 안의 ✔ 옆에 새로 생긴 Aa 버튼을 한 번 눌러보길. 텍스트를 선택한 상태에서 누르면 markup 도구가 펼쳐지고, 색상과 두께와 폰트가 그 자리에서 적용된다. Preview 토글을 눌러서 결과를 확인한 다음 ✓ Confirm. 이전엔 raw HTML을 머릿속으로 시뮬레이션하면서 적어야 했던 작업이, 이젠 시각적으로 한 번에 끝난다.
